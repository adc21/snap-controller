# snap-controller UI/UX 設計方針

## 最上位原則: .s8i が全ての起点

snap-controller の全機能は、**ユーザーが用意した .s8i ファイルに既に定義・配置されている
ダンパーを起点として動作する**。アプリが独自にダンパーを「ゼロから作る」ことはしない。

```
ユーザーの .s8i ファイル
  ├─ ダンパー定義 (DamperDefinition)
  │    DVOD / C1, 52, 1, 0, 0, 2, 0, 500, 150, 300, 10, 0.5, ...
  │    DSD  / S1, 0, 250000, 100, 200, 500, 0.02, ...
  │    DVMS / IRD1, ...  (iRDT/iOD)
  │
  └─ ダンパー配置 (DamperElement / RD)
       RD / D1, 101, 201, 1, C1, ..., 4    ← 節点101→201, 装置名C1, 基数4
       RD / D2, 102, 202, 1, C1, ..., 6    ← 節点102→202, 装置名C1, 基数6
```

全機能はこの .s8i を読み取り、表示・編集・最適化・設計・解析の起点とする。

## 機能別フロー

### 1. ケース編集（CaseEditDialog）

**.s8i の定義済みダンパーを読み取り、フィールド単位でパラメータを上書きする。**

```
.s8i → parse_s8i() → DamperDefinition 一覧 + DamperElement (RD) 一覧
  ↓
CaseEditDialog を開く
  ├─「ダンパー定義」タブ: 各定義の全フィールドを表示
  │   ユーザーが値を変更 → damper_params オーバーライドとして保存
  │   例: C1 の C0(減衰係数) を 500 → 800 に変更
  │
  └─「配置計画」タブ: 各RD要素の装置名・基数を表示
      ユーザーが基数を変更 → rd_overrides として保存
      例: D1 の基数を 4 → 6 に変更
  ↓
SNAP 実行時に .s8i のコピーにオーバーライドを適用 → 解析
```

- ダンパーの種類・フィールド名・単位は `damper_field_data.py` から取得
- 変更されたフィールドは黄色ハイライトで可視化
- 変更差分タブで元値と新値を一覧比較

### 2. iRDT設計ウィザード（IrdtWizardDialog）

**.s8i の固有値解析結果と構造データを起点に、iRDT（慣性質量ダンパー）を設計し、
結果を .s8i に書き戻す。**

```
.s8i → SNAP固有値解析 → 固有周期・刺激関数 → IrdtWizardDialog
  ↓
Step 1: .s8i の解析結果から対象モードを選択（周期・等価質量）
Step 2: 質量比 μ を設定 → Den Hartog / 拡張定点理論で最適同調
Step 3: 各階への配分方式を選択（モード形状ベース）
Step 4: FRF・感度解析チャートで設計結果を確認
Step 5: 出力先 .s8i を指定
  ↓
damper_injector.inject() で .s8i に DVMS 定義 + RD 要素を追加
  ↓
新しい AnalysisCase として登録 → SNAP実行で制振効果を検証
```

- Step 1 の固有周期は .s8i の固有値解析結果（.xbn）から読み取る
- Step 5 では元の .s8i を読み取り、ダンパー定義を**追加**して新 .s8i を出力
- 出力された .s8i は「iRDT が追加された状態」の新しいモデル

### 3. ダンパー挿入（DamperInjectorDialog）

**.s8i にiRDT / iOD を手動パラメータ指定で追加する。**

```
.s8i → DamperInjectorDialog
  ↓
ユーザーが各階のダンパーを手動指定:
  - ダンパー種類: iRDT（バネ付き）/ iOD（浮遊型、バネ=0）
  - 定義名, 階, 節点I/J, 基数, 慣性質量 m_d, バネ k_b, 減衰 c_d, ストローク
  ↓
damper_injector.inject() で .s8i に DVMS 定義 + RD 要素を追加
  ↓
新しい AnalysisCase として登録
```

- iRDT: 慣性質量ダンパー（バネ + ダッシュポット + inerter）
- iOD: 浮遊型大質量オイルダンパー（バネ = 0 の特殊ケース）
- 元の .s8i に既存のダンパーがあれば、それはそのまま残る（追加のみ）

### 4. 伝達関数解析（TransferFunctionWidget）

**.s8i に基づくSNAP解析の結果（時刻歴）を読み取り、周波数領域で解析する。**

```
.s8i → SNAP時刻歴解析 → 結果バイナリ (.hst)
  ↓
TransferFunctionWidget
  ├─ FFT振幅スペクトル: 床・層間・ダンパー・バネの応答をFFT
  └─ SNAP伝達関数: Welch法で H(f) = 出力/入力 を算出
  ↓
制振前/後のケースをリファレンス比較
  → ダンパーの制振効果を周波数領域で可視化
```

- 「ダンパー」カテゴリを選択すると、.s8i に定義されたダンパー要素の
  力・速度・ストロークの時刻歴をFFTできる
- リファレンス機能: 制振前ケースの伝達関数を基準線として保持し、
  ダンパー追加後の伝達関数と重ねて表示 → 応答低減効果を一目で確認

## 統合最適化ダイアログ（新規実装）

### 背景と動機

現在、ダンパーパラメータ最適化（OptimizerDialog）とダンパー基数最小化（MinimizerDialog）が
分離されている。実務では「本数を減らしつつ、残ったダンパーの特性を調整して性能を満たす」
といった**本数と特性の同時最適化**が求められる。

### コンセプト

**ユーザーの .s8i に定義済みのダンパーを起点に、パラメータと基数を同時に最適化する。**

### 全体フロー

```
1. ユーザーが .s8i ファイルを読み込む
      ↓
2. snap-controller が .s8i を解析し、定義済みダンパーを検出
   - ダンパー定義 (DamperDefinition): DVOD / C1, 52, 1, ...
   - ダンパー配置 (DamperElement/RD): 節点I, 節点J, 装置名, 基数
      ↓
3. 統合最適化ダイアログを開く
   - 検出されたダンパー定義のフィールドを一覧表示
     例: C0(減衰係数), Fc(リリーフ力), α(速度指数), 取付け剛性 ...
   - 各階のダンパー基数（RD要素のquantity）を一覧表示
     例: F3=4本, F5=6本, F7=2本 ...
      ↓
4. ユーザーが「変化させたいパラメータ」をチェックボックスで選択
   - 選択したパラメータの可動範囲（下限・上限・ステップ）を指定
   - チェックしないパラメータは現在値で固定
      ↓
5. 目的関数を最大2つ選択
   - 最大層間変形角, 最大加速度, せん断力係数, 総ダンパー本数 など
      ↓
6. 制約条件を設定
   - PerformanceCriteria（目標性能基準）から自動読み込み
      ↓
7. 探索アルゴリズムを選択・設定して最適化を開始
      ↓
8. snap-controller がアルゴリズムに従い自動的に解析ケースを生成
   - 選択パラメータの値をアルゴリズムが決定
   - .s8i のダンパー定義フィールドと RD 基数を書き換え
   - SNAP を実行して応答値（層間変形角・加速度等）を取得
      ↓
9. 結果をリアルタイムでプロットに追加
   - 2目的 → パレートフロント散布図
   - 1目的 → 収束プロット
   - ユーザーは実行中でもプロット上の点をクリックして候補の詳細を確認
      ↓
10. アルゴリズムが収束するか、ユーザーが停止するまで 8→9 を繰り返す
      ↓
11. 気に入った候補を選んで .s8i に適用
```

### 左ペイン: パラメータ・目的関数・制約設定

#### ダンパー定義パラメータ（.s8i から自動読み込み）

ダイアログ起動時に `.s8i` のダンパー定義を解析し、定義名とキーワードを表示する。

```
◆ ダンパー定義: "C1" (DVOD — 粘性/オイルダンパー)
```

そのダンパーの全フィールドを `get_damper_field_labels(keyword)` で取得し、
フィールド名・現在値・単位とともにテーブルに一覧表示する。

| ☑ | フィールド | 現在値 | 下限 | 上限 | ステップ | 単位 |
|---|-----------|--------|------|------|---------|------|
| ☑ | C0 (減衰係数) | 500 | 100 | 2000 | 50 | kN·s/mm |
| ☐ | Fc (リリーフ力) | 150 | — | — | — | kN |
| ☐ | Fy (最大減衰力) | 300 | — | — | — | kN |
| ☐ | Ve (基準速度) | 10 | — | — | — | mm/s |
| ☑ | α (速度指数) | 0.5 | 0.1 | 1.0 | 0.05 | — |
| ☐ | 剛性 | 1000 | — | — | — | kN/mm |
| ☐ | 取付け剛性 | 500 | — | — | — | kN/mm |

- **現在値**: .s8i の DamperDefinition.values[] から読み取った実際の値
- チェックを入れたフィールドだけが最適化変数になる
- チェックしないフィールドは現在値で固定
- 下限・上限は現在値を初期中央値として自動提案
- フィールドのインデックスは `damper_field_data.py` のマッピングに基づく

**複数ダンパー定義がある場合**:
ドロップダウンでダンパー定義を切り替え。
初期実装では「1つのダンパー定義」に対する最適化を対象とする。

#### ダンパー基数パラメータ（RD要素から自動読み込み）

`.s8i` の RD 要素を `build_floor_rd_map()` で解析し、
選択したダンパー定義に紐づく各階の基数（quantity）を表示する。

```
◆ ダンパー基数 (装置名 "C1" の配置)
```

| ☑ | 階 | 現在基数 | 下限 | 上限 | RD要素数 |
|---|-----|---------|------|------|---------|
| ☑ | F3 | 4 | 0 | 8 | 2箇所 |
| ☑ | F5 | 6 | 2 | 10 | 3箇所 |
| ☐ | F7 | 2 | — | — | 1箇所 |

- **現在基数**: RD 要素の quantity の合計
- **RD要素数**: その階に配置されている当該ダンパーの RD 要素の数
- 基数は整数型パラメータ（ステップ=1固定）
- 配置の**節点（node_i, node_j）は変化させない**（基数のみ）

#### 目的関数設定（最大2つ）

```
目的関数1: [最大層間変形角 ▼]  方向: [最小化 ▼]
目的関数2: [最大絶対加速度 ▼]  方向: [最小化 ▼]  ☐ 無効
```

- 1つ → 単目的最適化（全手法利用可能）
- 2つ → 多目的最適化（NSGA-II推奨、パレートフロント表示）
- 選択肢:

| キー | 表示名 | 説明 |
|------|--------|------|
| max_drift | 最大層間変形角 | 全階の最大値 |
| max_acc | 最大絶対加速度 | 全階の最大値 |
| max_disp | 最大相対変位 | 全階の最大値 |
| max_vel | 最大速度 | 全階の最大値 |
| shear_coeff | せん断力係数 | 最大値 |
| max_otm | 最大転倒モーメント | 基礎の最大値 |
| total_damper_count | 総ダンパー本数 | 全階合計 (SNAP実行不要) |

- `total_damper_count` はパラメータ値から直接計算（SNAP不要）
- 「性能を保ちつつ本数を最小化」→
  目的関数1=max_drift(最小化) + 目的関数2=total_damper_count(最小化)

#### 制約条件

- PerformanceCriteria から自動読み込み
- 制約違反の候補はプロット上で灰色×で表示

#### 探索手法の選択

| 目的数 | 推奨手法 | 利用可能手法 |
|--------|---------|------------|
| 1目的 | パラメータ数で自動推奨 | grid, random, LHS, bayesian, GA, SA, DE |
| 2目的 | NSGA-II | NSGA-II |

### 右ペイン: インタラクティブプロット

#### プロット仕様

- **2目的時**: X軸=目的関数1, Y軸=目的関数2 の散布図
- **1目的時**: X軸=反復番号, Y軸=目的関数値 の収束プロット
- **マーカー**: 青●=feasible, 灰×=infeasible, 橙●=パレートフロント, 赤★=最良解
- **リアルタイム更新**: SNAP実行完了ごとに点を追加

#### クリックインタラクション

- プロット上の点をクリック → 候補詳細パネルに情報表示
- ダブルクリック → 全パラメータ・全応答値の詳細ダイアログ
- 「この候補を .s8i に適用」ボタン（最適化途中でも適用可能）

## データフロー

### 評価関数の動作

```
アルゴリズムがパラメータ値を決定
  params = {"field_8": 850, "field_12": 0.75, "floor_count_F3": 4, "floor_count_F5": 6}
     ↓
1. .s8i ファイルをテンポラリにコピー
     ↓
2. 物理パラメータをダンパー定義に書き戻し
   - DamperDefinition "C1" の values[8] = "850"  (C0)
   - DamperDefinition "C1" の values[12] = "0.75" (α)
     ↓
3. 基数パラメータをRD要素に書き戻し
   - F3階のRD要素: quantity = 4
   - F5階のRD要素: quantity = 6
   （配置する節点は変えない。quantityのみ変更）
     ↓
4. SNAP.exe を実行
     ↓
5. 結果バイナリを読み込み → 応答値辞書を返す
   {"max_drift": 0.00553, "max_acc": 318, ..., "total_damper_count": 10}
```

### パラメータキーの命名規則

- **物理パラメータ**: `"field_{0-based index}"` → DamperDefinition.values[idx]
- **基数パラメータ**: `"floor_count_{階キー}"` → RD要素のquantity

## データモデル設計

### UnifiedOptimizationConfig

```python
@dataclass
class UnifiedOptimizationConfig:
    damper_def_name: str            # e.g. "C1"
    damper_keyword: str             # e.g. "DVOD"
    parameters: List[ParameterRange]
    objectives: List[ObjectiveSpec]  # max 2
    constraints: Dict[str, float]
    method: str
    max_iterations: int
    snap_timeout: int
    n_parallel: int
```

### ParameterRange 拡張

```python
@dataclass
class ParameterRange:
    key: str              # "field_8" or "floor_count_F3"
    display_name: str     # "C0 (減衰係数)" or "F3 ダンパー基数"
    min_val: float
    max_val: float
    step: float
    is_integer: bool      # 基数は True
    current_value: float  # .s8i から読み取った現在値
    field_index: int      # DamperDefinition.values の 0-based index (物理のみ)
    floor_key: str        # "F3" 等 (基数のみ)
    unit: str             # "kN·s/mm", "本" など
```

## UX原則

1. **.s8i が起点**: どの機能もダイアログを開いた瞬間に .s8i から読み取った
   ダンパー情報が表示される。ゼロからの設定は不要。
2. **選んで範囲を決めるだけ**: 最適化ではユーザーは変化させたいパラメータに
   チェックを入れ、可動範囲を指定するだけ。あとは snap-controller が自動で回す。
3. **設計者の判断を支援**: パレートフロントは「結論を出す」のではなく
   「設計者が判断材料を得る」ためのツール。
4. **途中停止の自由**: 良い候補が見つかったら最適化を途中で止めて適用できる。
5. **トレーサビリティ**: どのパラメータ設定でどの応答が得られたか、全履歴を保存。

## 実装フェーズ（統合最適化）

### Phase 1: 統合評価関数 (サービス層)
- SnapEvaluator を拡張し、物理パラメータと基数を同時に扱えるようにする
- `create_unified_evaluator()` 関数を新設
- 基数パラメータ（`floor_count_*`）→ build_floor_rd_map → RD要素quantity反映

### Phase 2: 統合最適化ダイアログ UI（左ペイン）
- `UnifiedOptimizerDialog` を新規作成
- .s8i からダンパー定義を読み取り、フィールド一覧+基数一覧をテーブル表示
- チェックボックスで最適化変数を選択、範囲を指定
- 目的関数（最大2つ）+ 制約条件 + 探索手法

### Phase 3: 統合最適化ダイアログ UI（右ペイン・インタラクティブプロット）
- パレートフロント / 収束プロット + リアルタイム更新
- クリックインタラクション + 候補詳細パネル + 途中適用ボタン
